## 3. 每条样本的复验动作

1. 先在业务模块补录源数据，不直接以大屏结果作为“写入成功”判断。
2. 通过 `/api/data-screen/overview` 或 `/api/data-screen/presales-focus-summary` 确认对应缺口计数下降。
3. 使用真实业务账号打开 `/data-screen`，确认预警提示、均值、加权合同池、工时汇总已变为可解释值。
4. 将“补录前截图 / 补录后截图 / 接口响应 / 数据库事实”并排存档。

## 4. 签字建议

1. 售前线先完成 `PS-01`、`PS-02`，确认工时问题已归零后再签售前视图。
2. 经营线先完成 `BS-01` 到 `BS-05` 这 5 条高影响样本，再决定是否进入 `BS-06` 到 `BS-10`。
3. `TD-01`、`TD-02` 不参与业务签字，只作为测试残留治理记录。
