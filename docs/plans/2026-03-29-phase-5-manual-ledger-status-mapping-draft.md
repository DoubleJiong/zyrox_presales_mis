## 1. 目的

本文件用于把人工 Excel 台账中的状态口径，映射到系统当前统一状态机口径，作为第五阶段业务对账前的人工确认基线。

本轮映射遵循以下原则：

1. 系统主业务阶段以 `projectStage` 为准。
2. `status` 仅作为兼容字段，不单独承载业务判断。
3. 如果人工台账无法精确区分两个相邻系统阶段，则先映射到“更保守、更靠前”的阶段，避免高估项目成熟度。
4. 本文件是对账草案，不等于自动导入规则；若业务侧确认后，可再固化为对账脚本或导入映射表。

## 2. 本轮确认的业务阶段口径

根据业务侧最新确认，第五阶段人工台账对账不采用系统当前更细的 10 段技术状态机作为业务展示口径，而采用 3 段业务主阶段：

1. `商机阶段`
2. `招投标阶段`
3. `归档阶段`

为兼容当前系统实现，本轮对账采用“两层口径”：

1. 业务对账口径：以上 3 个业务阶段。
2. 系统落点口径：仍需映射到系统现有 `projectStage` 字段中的某个技术阶段，以便后续对账落库。

本轮约定如下：

1. `商机阶段` 统一映射到系统 `opportunity`
2. `招投标阶段` 统一映射到系统 `bidding`
3. `归档阶段` 统一映射到系统 `archived`

说明：

1. 这不代表系统内部必须永久只保留 3 个技术阶段。
2. 这代表本轮人工台账对账时，业务上只认可 3 个主阶段。
3. 若后续正式决定把系统主阶段也收敛为 3 段，需要单独做状态机和统计口径调整，不在本次台账映射草案内直接变更代码。

## 5. 需要你确认的关键点

以下口径已由业务侧明确：

1. 系统业务阶段只按 `商机阶段`、`招投标阶段`、`归档阶段` 三段理解。
2. `交流`、`支持`、`常态化跟进`、`申报`、`控标`、`方案` 均属于 `商机阶段`。
3. 本表中没有稳定停留在 `招投标阶段` 的项目记录。
4. 一旦完成招投标，项目会很快进入 `归档阶段`。
5. `中标`、`签单`、`丢标` 属于 `归档阶段` 的状态表现。

## 6. 当前建议的确认版本

当前确认后的第五阶段第一轮业务对账口径如下：

1. `交流` / `支持` / `常态化跟进` / `强跟进` / `申报` / `控标` / `方案` -> `商机阶段`
2. `中标` / `已签单` / `丢标` / `放弃` -> `归档阶段`
3. 空值 -> 不纳入正式对账样本
4. `招投标阶段` 在本表中暂不形成有效样本，不作为本轮样本对账的必查集合

这样做的优点是：

1. 与业务侧真实管理习惯一致。
2. 不强行把手工台账塞进系统现有更细的技术阶段语义。
3. 足够支撑第五阶段的项目样本对账与正式评审前复核。

## 7. 下一步建议

在你确认本映射草案后，下一步按以下顺序推进：

1. 从 `项目跟进表` 选取重点项目样本。
2. 按本映射先转换成 3 段业务主阶段。
3. 再把 3 段业务主阶段折算到系统当前 `projectStage` 对账落点。
4. 与系统现有项目数据逐项比对：项目名称、客户、销售、阶段、金额、合同号。
5. 输出“匹配 / 不匹配 / 无法判断”三类差异清单。
