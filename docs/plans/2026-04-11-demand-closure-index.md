## 3. 推荐阅读顺序

### 场景 A：领导 / 管理口径汇报

1. 先看收敛结论。
2. 再看未闭环需求项闭环执行表。
3. 如需回答“下一轮做什么”，再看下一轮需求池。

### 场景 B：研发排期与建单

1. 先看下一轮需求池。
2. 再看主干需求研发任务卡。
3. 若排到第二梯队，直接看第二梯队需求研发任务卡。
4. 进入编码前，先看对应规则确认单。
5. 建单时直接使用对应的 Jira / 禅道可录入版。

### 场景 C：测试 / 业务验收执行

1. 先看待核需求项专项验收清单。
2. 当天执行时直接使用当日执行安排。
3. 若执行第二梯队，直接看第二梯队需求专项验收清单。
4. 若要先看第二梯队当前代码基线，直接看第二梯队规则验收基线。
5. 验收完成后回写收敛结论文档。
