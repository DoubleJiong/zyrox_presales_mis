# 正式投产验收与门禁清单

日期：2026-03-29

目标：为“试运行完成后是否允许正式投产”提供统一门禁，避免仅凭局部感觉或单一技术通过项做上线决策。

## 1. 使用原则

本清单用于：

1. 第五阶段正式投产评审。
2. 候选版本是否允许进入正式发布窗口的最终判定。

原则：

1. 四类门禁必须同时检查：业务、工程、安全、运维。
2. 任一阻断项未满足，默认不得正式投产。
3. 风险接受项必须有责任人、时限和补救计划。

## 2. 业务验收门禁

以下条件应全部满足：

1. 核心业务主链路已按统一状态机执行。
2. 审批流已按统一审批实体执行，且试点样本可追溯。
3. 方案主模型已按既定收敛策略落地，历史兼容策略清晰。
4. 仪表盘关键指标已完成样本对账。
5. 试点用户确认主要流程可用，无阻断业务缺陷。
6. 如系统设置或人员管理模块被纳入正式评审，必须按子模块级能力清单描述，不得用“系统设置整体可用”一类模糊措辞替代。

阻断项示例：

1. 项目阶段仍存在多套真相源。
2. 审批结果无法追溯到审批事件。
3. 关键指标与人工台账明显不一致且无解释。
4. 将“基础数据维护/恢复出厂设置”误表述为“备份恢复系统已完成”，导致正式评审对真实能力边界产生误判。

## 3. 工程验收门禁

以下条件应全部满足：

1. `typecheck`、`build`、核心测试、候选冒烟全部通过。
2. 候选版本来源清晰，构建产物与待发布代码一致。
3. CI 门禁已执行且无被绕过项。
4. 发布前检查、发布步骤、发布后观察项已分配责任人。
5. 试运行期间暴露的阻断级工程问题已关闭。

阻断项示例：

1. 构建不稳定或依赖人工补救。
2. 核心 E2E 冒烟不稳定。
3. 发布候选版本与测试版本不一致。

## 4. 安全验收门禁

以下条件应全部满足：

1. 无默认口令、无测试后门、无公开调试入口。
2. 无硬编码敏感配置，生产配置来源可控。
3. 登录、审批、密码重置、关键导出等敏感操作有审计记录。
4. 权限回归验证通过，未发现阻断级越权或漏权。
5. 风险接受项已登记并经负责人确认。

阻断项示例：

1. 仍可通过测试接口创建特权账号。
2. 仍存在默认密码或口令回退。
3. 关键权限角色可越权查看或修改业务数据。

## 5. 运维验收门禁

以下条件应全部满足：

1. 有可执行的部署手册和回滚手册。
2. 有备份与恢复演练记录，且恢复结果通过验证。
3. 有监控、日志和告警入口，能覆盖核心错误和关键链路。
4. 有发布窗口、观察期和值守安排。
5. 有故障升级路径和 on-call 责任人。

阻断项示例：

1. 没有验证过的恢复方案。
2. 发布后只能靠人工盯页面，没有告警与日志入口。
3. 没有回滚阈值和责任分工。

## 6. 投产评审输出物

正式投产评审至少应输出：

1. 门禁检查结果
2. 未完成项与风险接受项
3. 发布窗口与责任人
4. 观察时段与回滚触发条件
5. 评审结论：通过 / 有条件通过 / 不通过

## 7. 稳定观察期要求

正式投产不是发布完成即结束，还必须完成稳定观察期。至少确认：

1. 登录、项目主链路、审批、方案评审、仪表盘核心指标无阻断异常。
2. 无持续错误告警和异常峰值。
3. 试点用户无新增阻断问题。
4. 如存在一般问题，已进入后续治理台账且不影响正式投产结论。

## 8. 结论判定规则

### 可以正式投产

同时满足：

1. 四类门禁全部通过。
2. 阻断问题为零。
3. 发布窗口与回滚机制已就绪。

### 只能继续试运行，不得正式投产

出现任一情况：

1. 指标对账仍存在关键分歧。
2. 权限边界仍存在越权或漏权。
3. 恢复、回滚、监控任一运维核心能力未验证。

### 必须暂停并回到前一阶段

出现任一情况：

1. 业务规则本身仍未收敛。
2. 第三阶段核心改造后发现主链路设计错误。
3. 候选版本不具备基本发布稳定性。

本清单是“已完成试运行并正式投产”目标的最终门禁文档。后续执行时，应以本清单作为正式投产评审依据，而不是临时口头判断。
