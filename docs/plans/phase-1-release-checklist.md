# 第一阶段发布清单

适用范围：2026-03-29 第一阶段止血与安全兜底改造发布前检查。

## 1. 配置检查

1. `DATABASE_URL` 指向正确环境，且不使用占位值。
2. `JWT_SECRET` 已配置为目标环境专用密钥。
3. `NEXT_PUBLIC_APP_URL` 与目标访问域名一致。
4. `DEV_TEST_SETUP_ENABLED` 在目标环境为 `false`。
5. `TEST_SECRET` 不暴露在正式环境运行路径中。

## 2. 数据备份检查

1. 发布前已完成数据库备份。
2. 备份文件位置、时间和责任人已记录。
3. 若本次发布涉及权限或会话相关表，已确认恢复步骤可执行。

## 3. 构建产物检查

1. `corepack pnpm build` 在发布前通过。
2. 构建产物来自本次待发布代码版本。
3. 构建日志中不存在新的阻断级错误。

## 4. 权限冒烟检查

1. 未登录访问受保护接口返回 `401`。
2. 普通成员默认不能查看无负责人项目。
3. 普通成员默认不能上传合同敏感材料。
4. 非管理角色默认不应看到全局仪表盘数据。

## 5. 登录与注销检查

1. 登录页可正常渲染。
2. 登录后会话可建立。
3. 注销后旧 token 不可继续使用。
4. SSE 不再依赖 query token。

## 6. 核心接口冒烟检查

1. `GET /api/dashboard` 未登录返回 `401`。
2. `GET /login` 页面可访问并可交互。
3. `POST /api/contracts/upload` 在无权限场景返回 `403`。
4. `POST /api/auth/check-blacklist` 可正常响应。

## 7. 发布确认

1. 已确认回滚清单可执行。
2. 发布窗口、责任人和观察时段已明确。
3. 发布后首轮观察项已分配给具体责任人。
