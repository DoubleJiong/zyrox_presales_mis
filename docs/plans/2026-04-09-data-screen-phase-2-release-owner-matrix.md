# 数据大屏二期发布责任与回滚矩阵

日期：2026-04-09

适用范围：`/data-screen` 二期准发布、正式发布与回滚观察窗口。

复用依据：`docs/plans/2026-03-29-release-rollback-and-recovery-runbook.md`

## 1. 责任矩阵

| 责任项 | 当前责任岗 | 发布前需确认事项 |
| --- | --- | --- |
| 发布负责人 | 研发 / 发布执行人 | 负责发布窗口推进、发布单执行与最终口令 |
| 技术负责人 | 前端 / 全栈负责人 | 负责 runtime 健康检查、页面冒烟与异常兜底判断 |
| 数据库责任人 | DBA / 数据平台责任人 | 负责备份、迁移历史与必要时的数据恢复 |
| 业务观察人 | 市场部 / 售前负责人 | 负责观察窗口内业务读数与页面现象确认 |
| 回滚决策人 | 发布负责人 + 技术负责人 | 命中阈值时决定是否停止放量并回退 |

## 2. 观察窗口

1. 建议观察窗口：发布后 30 分钟。
2. 建议观察重点：登录页、`/data-screen` 一级入口、区域视角首屏、人员事项抽屉、专题风险抽屉、权限边界接口。

## 3. 回滚触发阈值

满足任一条件即触发回滚：

1. readiness 连续失败 3 次。
2. 登录主路径阻断超过 10 分钟。
3. `datascreen` 关键受保护接口持续返回 `5xx` 或页面一级入口不可用。
4. 观察窗口内无法通过小范围热修恢复主链路。

## 4. 当前结论

1. 回滚阈值与责任岗位已经落档，当前不再缺少 phase-2 专项发布口径。
2. 若进入正式发布，仅需在发布会前把“当前责任岗”补成实名即可，不需要再重建设计或流程。
