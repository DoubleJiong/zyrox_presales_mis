# 团队执行驾驶舱 V1 交付拆解

## 1. 文档目的

本文件在《团队执行驾驶舱 V1 方案》的基础上，进一步输出可排期、可拆分、可评估工作量的交付拆解，覆盖：

- 产品需求细化
- 页面模块级字段定义
- 后端读模型与接口拆分
- 权限与数据范围实施建议
- 建议迭代顺序

适合作为产品评审、技术评审和研发排期的中间版本。

## 2. V1 范围边界

## 2.1 纳入 V1

- 数据大屏体系下新增“团队执行驾驶舱”入口
- 驾驶舱只读分析页面
- 四类视角入口：角色、项目、客户、方案
- 核心指标区
- 风险聚焦区
- 视角分析区
- 详情穿透区
- 新增独立权限控制

## 2.2 不纳入 V1

- 页面内直接编辑/删除任务
- 页面内直接转派任务
- 页面内批量操作
- AI 建议与自动调配
- 大屏整体重构
- 新增复杂审批动作

## 3. 页面级需求拆解

## 3.1 统一页面骨架

### 页面名称

- 页面标题：团队执行驾驶舱
- 所属体系：数据大屏

### 页面级组件

- 顶部标题与说明区
- 视角切换栏
- 筛选条件栏
- 指标卡区
- 风险聚焦区
- 视角分析区
- 对象详情抽屉

### 通用交互

- 支持刷新数据
- 支持条件筛选后保留当前视角
- 支持点击图表元素穿透到详情
- 支持点击详情中的对象跳转原业务页面
- 所有图表和列表均支持空状态

## 3.2 顶部控制区需求

### 视角切换

必选四项：

- 角色视角
- 项目视角
- 客户视角
- 方案视角

### 筛选条件

建议首版统一提供：

- 时间范围
  - 今日
  - 近 7 天
  - 近 30 天
  - 自定义
- 组织范围
  - 全公司
  - 部门
  - 区域
  - 角色组
- 快速筛选
  - 仅看逾期
  - 仅看高优
  - 仅看重点项目
  - 仅看异常
- 搜索关键词
  - 人员
  - 项目
  - 客户
  - 方案

### V1 简化建议

若首版资源有限，建议分两层：

- 必做：视角切换、时间范围、异常筛选
- 可后置：组织范围、自定义时间、组合条件

## 3.3 核心指标区需求

### 指标卡定义

建议首版固定 8 张指标卡：

1. 待处理总量
2. 今日到期任务数
3. 已逾期任务数
4. 高优任务数
5. 当前活跃项目数
6. 重点项目涉及人员数
7. 负载过高人员数
8. 连续低活跃人员数

### 指标卡展示字段

每张卡建议包含：

- 指标名称
- 数值
- 同比或环比趋势（可后置）
- 颜色状态
- 点击后的穿透目标

### 指标卡点击行为

- 点击后联动风险区和分析表筛选
- 不直接跳页面，优先在当前页联动

## 3.4 风险聚焦区需求

### 区块组成

建议首版固定三列：

- 高风险人员榜
- 高风险项目榜
- 逾期/阻塞事项榜

### 每条记录字段

- 对象名称
- 风险等级
- 风险原因摘要
- 最近更新时间
- 跳转入口

### 风险定义建议

V1 采用规则型评分，不引入机器学习。

#### 人员高风险候选条件

- 未完成任务数超过阈值
- 逾期任务数超过阈值
- 同时参与多个重点项目
- 近 7 天推进动作明显不足

#### 项目高风险候选条件

- 项目进入关键阶段但任务逾期较多
- 关键任务集中于单一人员
- 最近推进时间过久
- 项目风险字段存在且未关闭

#### 事项高风险候选条件

- 逾期天数高
- 优先级高且未开始
- 被多个对象关联但长期未关闭

## 3.5 详情穿透区需求

### 触发方式

- 点击图表柱、点、卡片或榜单行
- 点击分析表中的对象名称

### 详情抽屉内容结构

#### 对象基本信息

- 名称
- 所属视角类型
- 当前状态
- 责任人/负责人
- 最近更新时间

#### 当前执行摘要

- 未完成任务数
- 逾期任务数
- 高优任务数
- 当前待办数
- 预警数

#### 关联对象信息

- 参与项目
- 关联客户
- 关联方案
- 参与人员

#### 最近活动时间线

- 最近任务更新
- 最近待办变更
- 最近项目推进
- 最近客户互动或方案更新

#### 操作入口

- 打开任务中心
- 打开项目详情
- 打开客户详情
- 打开方案详情

## 5. 后端实施拆解

## 5.1 实施原则

- 不直接复用任务中心页面接口
- 不通过前端拼装多个业务接口完成驾驶舱
- 建立独立读模型和独立聚合 API

## 5.2 建议读模型结构

建议新增 `cockpit` 读模型目录，示例：

- `src/lib/cockpit/read-model.ts`
- `src/lib/cockpit/risk-rules.ts`
- `src/lib/cockpit/serializers.ts`

### 建议读模型层级

#### 第一层：统一摘要读模型

负责：

- 指标卡
- 风险榜单
- 当前筛选条件下的摘要

#### 第二层：视角读模型

负责：

- 角色视角聚合
- 项目视角聚合
- 客户视角聚合
- 方案视角聚合

#### 第三层：详情读模型

负责：

- 指定对象详情
- 最近活动时间线
- 关联对象摘要

## 5.3 API 拆分建议

### 首页摘要

- `GET /api/cockpit/summary`

返回：

- 核心指标卡数据
- 风险榜单数据
- 当前可用筛选范围

### 视角查询

- `GET /api/cockpit/views/roles`
- `GET /api/cockpit/views/projects`
- `GET /api/cockpit/views/customers`
- `GET /api/cockpit/views/solutions`

返回：

- 图表数据
- 明细表数据
- 分页信息

### 详情接口

- `GET /api/cockpit/detail/role/{id}`
- `GET /api/cockpit/detail/project/{id}`
- `GET /api/cockpit/detail/customer/{id}`
- `GET /api/cockpit/detail/solution/{id}`

返回：

- 基本信息
- 执行摘要
- 关联对象摘要
- 最近活动时间线

## 6. 权限与数据范围实施拆解

## 6.1 功能权限

建议新增权限常量：

- `cockpit:view`

如需进一步细分，也可拆成：

- `cockpit:view`
- `cockpit:export`

但 V1 建议至少先有 `cockpit:view`。

## 6.2 数据范围控制

建议在角色数据权限中新增资源类型：

- `cockpit`

支持范围：

- `self`
- `manage`
- `role`
- `all`

### 范围解释建议

- `self`：只能看到与本人有关的对象汇总
- `manage`：只能看到自己管理范围内人员和对象
- `role`：只能看到同角色或角色组范围
- `all`：可见全公司

## 6.3 前端权限表现

- 无权限用户不显示入口
- 有入口无数据范围时返回空态，不报错
- 无权限直接访问返回 403 或重定向

## 9. 当前建议的最小可交付版本

如果必须进一步压缩首版范围，建议最小可交付版本为：

- 数据大屏入口
- 驾驶舱页面骨架
- 统一指标区
- 风险榜单
- 角色视角
- 项目视角
- 详情抽屉
- 独立权限控制

客户视角和方案视角可保留入口说明，但延后完整实现。

## 10. 结论

下一步如果进入研发阶段，建议先按“骨架与权限 -> 摘要与角色视角 -> 项目视角 -> 客户与方案视角”的顺序推进。

从当前系统基础看，最稳妥的首版落点不是“一次做全”，而是优先把：

- 入口位置
- 页面边界
- 权限模型
- 团队级聚合读模型
- 角色与项目两类主视角

这五部分先跑通。
