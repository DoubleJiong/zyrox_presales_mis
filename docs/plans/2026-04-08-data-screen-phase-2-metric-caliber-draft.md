# 2026-04-08 数据大屏二期指标口径清单（草案）

## 1. 文档目的

本文档用于明确数据大屏二期中“区域视角”和“人员视角”的核心指标口径，避免后续出现：

1. 页面上有图，但图和业务理解不一致。
2. 区域视角与人员视角统计口径冲突。
3. 全国层级与浙江层级显示逻辑不一致。
4. 大屏和其他模块对同一指标的理解不一致。

本文档当前为草案，目标是先建立统一的指标语言，后续再进入数据源确认、SQL 实现、接口出参定义和验收清单。

## 2. 总体原则

### 2.1 视角并列，指标分层

二期指标设计遵循以下结构：

1. 区域视角：区域 -> 各类统计口径
2. 人员视角：人员 -> 事项及各类统计口径

两条视角共享底层业务对象，但不强行使用相同展示指标。

### 2.2 指标分类

所有指标建议拆成 4 类：

1. 规模类：总量、数量、金额
2. 结构类：分布、构成、阶段占比
3. 过程类：趋势、活跃度、推进速度
4. 风险类：逾期、阻塞、异常、低活跃

### 2.3 口径优先级

建议口径分成三个优先级：

1. P0：大屏主层必须有
2. P1：大屏下钻层应有
3. P2：后续增强或专题层补充

## 3. 业务对象定义

二期指标主要围绕以下业务对象展开：

1. 客户
2. 项目
3. 任务
4. 待办
5. 方案
6. 售前活动
7. 人员
8. 合同

说明：
当前若部分对象尚无完整结构数据，可先在口径层定义，再在实现阶段标注“待补数据源”。

## 4. 区域视角指标口径

## 4.1 区域视角目标

区域视角的指标用于回答：

1. 各区域业务盘子大小如何。
2. 各区域业务结构是否均衡。
3. 各区域推进活跃度如何。
4. 各区域风险压力在哪里。

区域视角不以“人”为核心，而以“区域业务状态”为核心。

## 4.2 区域基础维度

区域视角统一使用两层空间维度：

1. 全国层：按省份统计
2. 浙江层：按地市统计

要求：

1. 全国层与浙江层必须使用同一套指标定义。
2. 浙江层只是空间颗粒度更细，不是另一套完全不同的统计体系。

## 4.3 区域视角 P0 指标

### 4.3.1 客户总量

- 指标名称：客户总量
- 指标分类：规模类
- 统计对象：客户
- 口径定义：当前归属到该区域的客户数量
- 推荐时间属性：当前时点
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.2 活跃客户数

- 指标名称：活跃客户数
- 指标分类：过程类
- 统计对象：客户
- 口径定义：在选定时间范围内，有项目、售前活动、方案、任务推进痕迹的客户数量
- 推荐时间属性：近 7 天 / 近 30 天
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.3 项目总量

- 指标名称：项目总量
- 指标分类：规模类
- 统计对象：项目
- 口径定义：当前归属到该区域的项目数量
- 推荐时间属性：当前时点
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.4 活跃项目数

- 指标名称：活跃项目数
- 指标分类：过程类
- 统计对象：项目
- 口径定义：在选定时间范围内仍有推进动作且状态未结束的项目数量
- 推荐时间属性：近 7 天 / 近 30 天
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.5 项目金额

- 指标名称：项目金额
- 指标分类：规模类
- 统计对象：项目
- 口径定义：归属到该区域项目的累计金额或当前有效金额总和
- 推荐时间属性：当前时点 + 时间筛选
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.6 合同金额

- 指标名称：合同金额
- 指标分类：规模类
- 统计对象：合同
- 口径定义：归属到该区域合同的累计金额
- 推荐时间属性：当前时点 + 时间筛选
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.7 售前活动数

- 指标名称：售前活动数
- 指标分类：过程类
- 统计对象：售前活动
- 口径定义：选定时间范围内发生在该区域或服务于该区域客户/项目的售前活动数量
- 推荐时间属性：近 7 天 / 近 30 天
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.8 方案支撑数

- 指标名称：方案支撑数
- 指标分类：过程类
- 统计对象：方案
- 口径定义：选定时间范围内与该区域客户或项目关联的方案数量
- 推荐时间属性：近 7 天 / 近 30 天
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.9 风险项目数

- 指标名称：风险项目数
- 指标分类：风险类
- 统计对象：项目
- 口径定义：满足高风险阈值的项目数量
- 推荐时间属性：当前时点 + 时间筛选
- 适用层级：全国、浙江
- 优先级：P0

### 4.3.10 异常事项数

- 指标名称：异常事项数
- 指标分类：风险类
- 统计对象：任务 + 待办 + 评审事项
- 口径定义：在选定时间范围内，归属到该区域且处于逾期、阻塞、高优未推进状态的事项数量
- 推荐时间属性：近 7 天 / 近 30 天
- 适用层级：全国、浙江
- 优先级：P0

## 4.4 区域视角 P1 指标

### 4.4.1 客户类型结构

- 指标名称：客户类型结构
- 指标分类：结构类
- 统计对象：客户
- 口径定义：该区域客户按类型划分后的数量占比
- 优先级：P1

### 4.4.2 项目阶段结构

- 指标名称：项目阶段结构
- 指标分类：结构类
- 统计对象：项目
- 口径定义：该区域项目按阶段划分后的数量占比
- 优先级：P1

### 4.4.3 方案状态结构

- 指标名称：方案状态结构
- 指标分类：结构类
- 统计对象：方案
- 口径定义：该区域方案按状态划分后的数量占比
- 优先级：P1

### 4.4.4 风险等级结构

- 指标名称：风险等级结构
- 指标分类：结构类
- 统计对象：风险对象
- 口径定义：按高风险、中风险、一般风险划分的对象占比
- 优先级：P1

### 4.4.5 区域推进趋势

- 指标名称：区域推进趋势
- 指标分类：过程类
- 统计对象：客户、项目、活动、方案
- 口径定义：按时间序列统计该区域在所选口径下的变化趋势
- 优先级：P1

## 5. 人员视角指标口径

## 5.1 人员视角目标

人员视角的指标用于回答：

1. 团队整体负载是否失衡。
2. 哪些人工作过载。
3. 哪些人推进不足。
4. 每个人具体在做什么事项。

人员视角的核心不是“区域中的人”，而是“人员及其事项”。

## 5.2 人员基础维度

人员视角建议至少支持以下维度：

1. 人员
2. 角色组
3. 部门
4. 区域归属（仅作为过滤条件，不作为主叙事）
5. 时间范围

## 5.3 人员视角 P0 指标

### 5.3.1 管理中人员数

- 指标名称：管理中人员数
- 指标分类：规模类
- 统计对象：人员
- 口径定义：当前纳入管理范围且处于有效状态的人员数量
- 优先级：P0

### 5.3.2 当前任务总量

- 指标名称：当前任务总量
- 指标分类：规模类
- 统计对象：任务
- 口径定义：当前未完成任务数量
- 优先级：P0

### 5.3.3 当前待办总量

- 指标名称：当前待办总量
- 指标分类：规模类
- 统计对象：待办
- 口径定义：当前未完成待办数量
- 优先级：P0

### 5.3.4 当前事项总量

- 指标名称：当前事项总量
- 指标分类：规模类
- 统计对象：任务 + 待办 + 评审事项
- 口径定义：当前全部未关闭事项总量
- 优先级：P0

### 5.3.5 逾期事项数

- 指标名称：逾期事项数
- 指标分类：风险类
- 统计对象：任务 + 待办 + 评审事项
- 口径定义：截止时间已过且当前仍未关闭的事项数量
- 优先级：P0

### 5.3.6 高优事项数

- 指标名称：高优事项数
- 指标分类：风险类
- 统计对象：事项
- 口径定义：优先级为 high 或 urgent 的未完成事项数量
- 优先级：P0

### 5.3.7 过载人员数

- 指标名称：过载人员数
- 指标分类：风险类
- 统计对象：人员
- 口径定义：事项量、逾期量或高优事项量达到阈值的人员数量
- 优先级：P0

### 5.3.8 连续低活跃人员数

- 指标名称：连续低活跃人员数
- 指标分类：风险类
- 统计对象：人员
- 口径定义：在设定时间阈值内缺少任务、项目、方案、活动推进痕迹的人员数量
- 优先级：P0

### 5.3.9 当前参与项目数

- 指标名称：当前参与项目数
- 指标分类：过程类
- 统计对象：人员与项目关系
- 口径定义：当前处于活跃状态、且该人员参与中的项目数量
- 优先级：P0

### 5.3.10 当前参与方案数

- 指标名称：当前参与方案数
- 指标分类：过程类
- 统计对象：人员与方案关系
- 口径定义：当前该人员参与中的方案数量
- 优先级：P0

## 5.4 人员视角 P1 指标

### 5.4.1 角色组负载分布

- 指标名称：角色组负载分布
- 指标分类：结构类
- 统计对象：角色组
- 口径定义：按角色组汇总待处理量、逾期量、高优量的分布情况
- 优先级：P1

### 5.4.2 人员风险分

- 指标名称：人员风险分
- 指标分类：风险类
- 统计对象：人员
- 口径定义：综合事项量、逾期量、高优事项量、活跃度等因素形成的风险评分
- 优先级：P1

### 5.4.3 人员协同密度

- 指标名称：人员协同密度
- 指标分类：过程类
- 统计对象：人员关系
- 口径定义：在事项、项目、方案协作中形成的协同关系强度
- 优先级：P1

### 5.4.4 人员事项趋势

- 指标名称：人员事项趋势
- 指标分类：过程类
- 统计对象：事项
- 口径定义：按时间序列统计该人员事项新增、完成、逾期变化趋势
- 优先级：P1

## 5.5 人员视角 P0 事项层字段

人员视角最终必须落到事项级，因此以下字段建议列为 P0：

1. 事项名称
2. 事项类型
3. 所属客户
4. 所属项目
5. 所属方案
6. 当前负责人
7. 协同对象
8. 当前状态
9. 优先级
10. 截止时间
11. 是否逾期
12. 最近推进时间
13. 当前阻塞原因

## 6. 区域与人员视角共用指标映射

虽然区域与人员是并列视角，但有些指标来源相同，只是聚合方式不同。建议统一维护以下映射关系：

1. 事项总量：
   - 区域视角按区域聚合
   - 人员视角按人员聚合
2. 高优事项数：
   - 区域视角按区域聚合
   - 人员视角按人员聚合
3. 风险对象数：
   - 区域视角看区域风险面
   - 人员视角看人员风险面
4. 活跃度：
   - 区域视角看区域活跃度
   - 人员视角看个人活跃度

## 7. 浙江地图专项指标要求

浙江层级必须至少支持以下 P0 指标真实落地：

1. 客户总量
2. 活跃客户数
3. 项目总量
4. 活跃项目数
5. 项目金额
6. 合同金额
7. 售前活动数
8. 方案支撑数
9. 风险项目数
10. 异常事项数

如暂时无法全部落地，必须明确标注：

1. 数据源缺失
2. 数据未接入
3. 仅前端未渲染
4. 暂无真实数据

## 8. 当前与未来模块关系备注

后续若将“绩效管理”的人员统计结果逐步并入大屏，则这些绩效统计应优先映射到人员视角指标体系，而不是新增第三套人员统计口径。

当前建议：

1. 大屏人员视角负责“人员统计结果展示”。
2. 绩效管理模块未来只保留绩效考核模型、规则、配置与考核流程。

本条仅作为未来演进约束，不纳入当前交付范围。
