# 2026-04-11 第二梯队需求专项验收清单

## 1. 目标

本文件用于对第二梯队三项需求发起开发前规则验收和开发后功能验收。

本清单的输出目标只有两个：

1. 形成“规则已收敛，可进入开发”的结论。
2. 形成“功能已通过，可关闭”的结论。

不再保留“可以做，但规则还靠口头补充”的中间状态。

## 2. 适用范围

本轮第二梯队包括：

1. 附加项 B：周报 / 周报导出。
2. 附加项 C：工作台标签收拢到工作台下方。
3. `1.1`：客户关键字去重提醒。

## 3. 验收统一规则

### 3.1 验收记录要求

每项验收至少保留以下证据：

1. 页面入口。
2. 验收账号。
3. 业务样本。
4. 页面截图或录屏。
5. 最终结论。

### 3.2 验收结论规则

每项必须经历两个阶段：

1. 规则验收：确认业务规则收敛，可进入开发。
2. 功能验收：确认实现结果符合规则，可进入关闭。

若规则验收未通过，则不得直接进入编码实施。

## 4. 分项专项验收脚本

### 4.1 附加项 B：周报 / 周报导出

#### 4.1.1 规则验收目标

确认周报能力已具备明确的统计口径、权限边界、入口位置和导出字段定义。

#### 4.1.2 页面入口

1. 工作台或报表入口。
2. 周报页面或周报弹层。
3. 周报导出入口。

#### 4.1.3 验收样本

1. 至少 1 个管理视角账号。
2. 至少 1 个普通视角账号。
3. 至少 1 组有数据周样本。
4. 至少 1 组无数据周样本。

#### 4.1.4 规则检查点

1. 周报统计周期是自然周、业务周还是自定义周。
2. 周报统计字段是否有明确列头、顺序和口径说明。
3. 管理视角与普通视角的查看范围、导出范围是否不同。
4. 空数据、跨周、跨月周的展示规则是否明确。

#### 4.1.5 规则通过标准

1. 周报字段、周期、筛选条件可形成文档化说明。
2. 权限边界明确。
3. 导出字段与页面预览口径一致。

#### 4.1.6 功能检查点

1. 用户能从明确入口进入周报页面。
2. 生成周报时有加载反馈。
3. 无数据和失败态可识别。
4. 导出文件可成功下载，列头与页面预览一致。
5. 无权限账号不会看到导出入口或会被正确拦截。

#### 4.1.7 功能通过标准

1. 周报生成与导出完整可用。
2. 页面结果、导出结果与规则定义一致。
3. 权限控制符合预期。

### 4.2 附加项 C：工作台标签收拢到工作台下方

#### 4.2.1 规则验收目标

确认工作台标签区的新落点、信息层级、分组方式和响应式规则已经收敛。

#### 4.2.2 页面入口

1. 工作台首页。
2. 原标签入口所在区域。
3. 标签点击后的目标页面。

#### 4.2.3 验收样本

1. 至少 1 个标签较少场景。
2. 至少 1 个标签较多场景。
3. 至少覆盖 2 种桌面分辨率。

#### 4.2.4 规则检查点

1. 标签区放在工作台哪一块内容之下。
2. 标签区与摘要卡片、快捷入口、图表区域的先后关系。
3. 标签数量较多时的折行、折叠或滚动规则。
4. 标签文案、排序和分组是否有明确口径。

#### 4.2.5 规则通过标准

1. 工作台标签落点明确。
2. 信息层级与视觉优先级清晰。
3. 多标签场景与常见分辨率处理方式明确。

#### 4.2.6 功能检查点

1. 标签区已迁移至目标位置。
2. 标签入口可见性不下降。
3. 标签点击跳转行为正常。
4. 页面在常见桌面分辨率下无明显遮挡、错位、抖动。

#### 4.2.7 功能通过标准

1. 标签迁移结果符合落点定义。
2. 交互行为未回退。
3. 页面结构和信息层级更稳定。

### 4.3 `1.1` 客户关键字去重提醒

#### 4.3.1 规则验收目标

确认客户去重提醒的匹配规则、提示时机、阻断策略和相似客户展示字段已经收敛。

#### 4.3.2 页面入口

1. 客户新建页或新建客户弹窗。
2. 客户编辑页。
3. 相似客户查看入口。

#### 4.3.3 验收样本

1. 1 个完全重复客户名称样本。
2. 1 个简称或关键字相似样本。
3. 1 个仅部分包含但应允许提交样本。
4. 1 个正常无匹配样本。

#### 4.3.4 规则检查点

1. 哪些字段参与匹配：客户名称、简称、关键字或其他字段。
2. 何时触发提醒：输入中、失焦后、保存前或提交时。
3. 是强阻断还是提醒后允许继续提交。
4. 相似客户列表至少展示哪些辅助判断字段。

#### 4.3.5 规则通过标准

1. 匹配规则可复测。
2. 提示时机清晰。
3. 阻断与放行边界明确。

#### 4.3.6 功能检查点

1. 完全重复样本会触发提醒或阻断。
2. 相似样本会展示辅助判断信息。
3. 允许继续提交的例外场景可正常放行。
4. 无匹配样本录入链路不受影响。

#### 4.3.7 功能通过标准

1. 去重提醒能有效暴露潜在重复客户。
2. 提醒信息足以支持业务判断。
3. 不误伤正常录入流程。

## 5. 执行顺序建议

建议按以下顺序发起专项验收：

1. 周报统计口径与导出规则。
2. 工作台标签落点与信息层级。
3. 客户关键字去重规则。
4. 周报功能验收。
5. 工作台标签功能验收。
6. 客户去重提醒功能验收。

原因：先把规则收敛，再进实现验收，能减少返工。

## 6. 验收后的台账改写动作

专项验收完成后，必须同步执行以下改写：

1. 在《下一轮需求池清单》中把对应项从“可排期”改成“开发中”或“已关闭”。
2. 在《未闭环需求项闭环执行表》中更新当前动作和退出标准完成情况。
3. 在《第二梯队需求研发任务卡》或《第二梯队需求建单精简版》中回写完成状态。
4. 在《需求闭环文档总索引》中补充最终入口说明。
