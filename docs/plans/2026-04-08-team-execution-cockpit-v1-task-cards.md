# 团队执行驾驶舱 V1 开发任务卡

日期：2026-04-08

文档目标：把《团队执行驾驶舱 V1 研发任务清单》继续转成可直接录入项目管理系统的开发任务卡。

适用方式：

1. 每张任务卡可直接对应 1 个 issue 或 1 个研发任务单。
2. 若团队人数较少，可把同角色的相邻任务卡合并执行。
3. 默认按 MVP 优先级推进，先保证首屏、角色视角、项目视角上线。

## 1. 使用原则

### 卡片状态建议

- `todo`：未开始
- `doing`：开发中
- `review`：待评审/待联调
- `qa`：待测试
- `done`：已完成

### 优先级建议

- `P0`：MVP 必做
- `P1`：V1 建议做
- `P2`：可后置

### 预估方式建议

- `XS`：0.5 天以内
- `S`：1 天
- `M`：2 至 3 天
- `L`：4 至 5 天

## 2. 建议建卡顺序

建议按以下顺序录入任务系统：

1. TC-01 至 TC-04

## 3. 任务卡明细

## TC-01 权限常量与访问边界

**任务类型：** 后端 / 权限

**优先级：** P0

**建议工期：** S

**建议负责人：** 后端或全栈

**前置依赖：** 无

**任务目标：** 为团队执行驾驶舱建立独立权限常量、路由访问控制和角色配置入口。

**涉及文件（预计）：**

- `src/lib/permissions.ts`
- `src/app/settings/roles/page.tsx`
- 驾驶舱路由权限判断相关文件

**交付物：**

- 新的驾驶舱权限常量
- 路由访问控制接入
- 角色权限页可配置项

**完成定义：**

- 有权限用户可以访问驾驶舱
- 无权限用户不可见入口，直接访问也会被拦截
- 不破坏现有数据大屏和任务中心权限逻辑

**验收要点：**

- 正常账号、无权限账号各验证 1 次
- 角色权限页勾选后生效

## TC-02 导航入口与路由骨架

**任务类型：** 前端

**优先级：** P0

**建议工期：** S

**建议负责人：** 前端

**前置依赖：** TC-01

**任务目标：** 建立驾驶舱入口、页面路由和空白骨架页。

**涉及文件（预计）：**

- `src/components/navigation-menu.tsx`
- `src/components/mobile-nav.tsx`
- `src/app/data-screen/**`

**交付物：**

- 驾驶舱页面路由
- 数据大屏体系入口
- 页面基础标题和说明区

**完成定义：**

- 桌面端导航可见并正确跳转
- 移动端行为有明确处理，显示或隐藏均可，但口径一致
- 页面能以空骨架形式稳定打开

**验收要点：**

- 桌面导航链路通
- 刷新路由无报错
- 无权限时不暴露入口

## TC-03 统一筛选模型与查询协议

**任务类型：** 前端 / 后端协同

**优先级：** P0

**建议工期：** M

**建议负责人：** 全栈

**前置依赖：** TC-01

**任务目标：** 为页面顶部控制区建立统一筛选状态、URL 参数协议和接口查询参数结构。

**涉及文件（预计）：**

- 驾驶舱筛选组件
- 驾驶舱页面参数解析工具
- 驾驶舱 API 参数校验工具

**交付物：**

- 筛选参数类型定义
- 默认视角和默认时间范围
- URL 查询参数协议
- 前后端统一参数映射

**完成定义：**

- 切换视角后保留基础筛选值
- 刷新页面后主要筛选状态可恢复
- 接口收到的查询结构统一且可校验

**验收要点：**

- 至少验证视角、时间范围、异常筛选、关键词 4 类参数
- 非法参数有兜底处理
