# 数据大屏二期开发任务卡

日期：2026-04-08

文档目标：把《数据大屏二期研发任务拆解》继续转成可直接录入项目管理系统的开发任务卡。

适用方式：

1. 每张任务卡可直接对应 1 个 issue 或 1 个研发任务单。
2. 若团队人数有限，可把相邻且同角色主责的任务卡合并执行。
3. 默认按 MVP 优先级推进，先保证单入口、浙江止血、区域视角、人员视角、统一下钻层可用。

## 1. 使用原则

### 卡片状态建议

- `todo`：未开始
- `doing`：开发中
- `review`：待评审或待联调
- `qa`：待测试
- `done`：已完成

### 优先级建议

- `P0`：MVP 必做
- `P1`：二期建议做
- `P2`：可后置

### 预估方式建议

- `XS`：0.5 天以内
- `S`：1 天
- `M`：2 至 3 天
- `L`：4 至 5 天

## 2. 建议建卡顺序

建议按以下顺序录入任务系统：

1. DTC-01 至 DTC-03
2. DTC-04 至 DTC-06
3. DTC-07 至 DTC-10
4. DTC-12 至 DTC-15
5. DTC-11 视资源情况插入或后置

其中 MVP 最小上线范围建议为：

- DTC-01 导航与信息架构收口
- DTC-02 统一筛选模型与 URL 协议
- DTC-03 浙江地图数据链路止血
- DTC-04 区域视角初始化读模型
- DTC-05 区域视角页面首屏重构
- DTC-07 人员视角初始化读模型
- DTC-08 人员视角页面首屏重构
- DTC-09 人员事项穿透与异常筛选
- DTC-10 统一下钻层与业务跳转
- DTC-12 图表组件收敛与视觉统一
- DTC-14 自动化测试与业务验收
- DTC-15 联调收口与发布准备

## 3. 任务卡明细

## DTC-01 导航与信息架构收口

**任务类型：** 前端

**优先级：** P0

**建议工期：** S

**建议负责人：** 前端

**前置依赖：** 无

**任务目标：** 把“数据大屏”和“团队驾驶舱”收敛为单入口，并建立二期统一页面骨架。

**涉及文件（预计）：**

- `src/components/navigation-menu.tsx`
- `src/components/mobile-nav.tsx`
- `src/app/data-screen/page.tsx`
- 数据大屏视角切换相关组件

**交付物：**

- 单入口导航
- 数据大屏二期页面骨架
- 一级视角切换位

**完成定义：**

- 导航中只保留“数据大屏”统一入口
- 页面内部可切换区域视角、人员视角、专题视角
- 不破坏现有 `/data-screen` canonical 路由

**验收要点：**

- 桌面端和移动端入口口径一致
- 刷新和直达链接无报错
- 无权限时不暴露错误入口

## DTC-02 统一筛选模型与 URL 协议

**任务类型：** 前后端协同

**优先级：** P0

**建议工期：** M

**建议负责人：** 全栈

**前置依赖：** DTC-01

**任务目标：** 为区域视角和人员视角建立共享筛选模型、URL 参数协议和接口参数校验层。

**涉及文件（预计）：**

- 数据大屏筛选组件
- 页面参数解析工具
- API 参数校验工具
- 数据大屏 hook 或 query model 相关文件

**交付物：**

- 公共筛选参数类型定义
- 视角专属筛选参数定义
- URL 查询参数协议
- 前后端统一参数映射

**完成定义：**

- 切换视角时公共筛选保留，专属筛选按规则重置
- 刷新页面后主要筛选状态可恢复
- 接口收到的查询结构统一且可校验

**验收要点：**

- 至少验证视角、时间范围、区域、异常筛选 4 类参数
- 非法参数有兜底处理
- 前端不再各自维护多套筛选状态

## DTC-03 浙江地图数据链路止血

**任务类型：** 前后端协同

**优先级：** P0

**建议工期：** M

**建议负责人：** 全栈

**前置依赖：** DTC-02

**涉及文件（预计）：**

- `src/lib/data-screen-map.ts`
- `src/app/api/data-screen/heatmap/route.ts`
- 浙江地图相关图表或排行组件
- 浙江专项问题清单文档

**交付物：**

- 浙江真实取数链路修复
- 浙江最小可用指标集
- 浙江专项问题清单

**完成定义：**

- 浙江地图不再强制返回全零默认值

- 未建设指标不再静默填零，而是显式标注状态

**验收要点：**

- 地图颜色和排行能体现真实差异
- 业务能区分“无数据”和“未建设”
- 浙江模式切换不再出现误导性空白结果

## 4. 录入任务系统建议字段

建议在任务系统中统一补充以下字段：

1. `模块`：数据大屏二期
2. `子模块`：区域视角 / 人员视角 / 统一下钻 / 浙江专项 / 验收发布
3. `优先级`：P0 / P1 / P2
4. `状态`：todo / doing / review / qa / done
5. `工期`：XS / S / M / L
6. `里程碑`：M1 / M2 / M3 / M4

## 5. 补充提醒

1. 浙江专项不建议作为普通缺陷夹在区域视角任务里，应该独立建卡跟踪，否则很容易被页面开发节奏掩盖。
2. 人员事项对象模型建议尽早冻结最小字段集，否则 DTC-08 和 DTC-09 容易反复返工。
3. 统一下钻层不要等所有页面完成后再做，否则区域和人员会各自长出一套交互壳层。
4. 绩效统计并入大屏是后续方向，本期只保留插槽，不纳入当前卡片范围。
