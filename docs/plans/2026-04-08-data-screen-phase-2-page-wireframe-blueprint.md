# 2026-04-08 数据大屏二期页面线框图级文字蓝图

## 1. 文档目的

本文档在《数据大屏二期信息架构与页面分区草案》基础上，继续向前推进一层，明确二期页面的线框图级结构。

本蓝图不关注视觉细节，不限定最终颜色、动效与组件技术实现，而聚焦以下问题：

1. 每个视角页面到底分成哪几块。
2. 每块放什么内容。
3. 每块优先用什么图表表达。
4. 用户从哪里点击、如何继续下钻。

本文档用于指导后续：

1. 原型设计
2. 视觉设计
3. 前端页面重构
4. 后端接口口径对齐

## 2. 页面总体框架

数据大屏二期统一维持一个页面入口，即“数据大屏”。

页面内部固定使用如下总框架：

1. 顶部统一控制区
2. 一级视角切换区
3. 主内容区
4. 下钻分析层

可抽象为：

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部统一控制区                                              │
├──────────────────────────────────────────────────────────────┤
│ 一级视角切换：区域视角 | 人员视角 | 专题视角                │
├──────────────────────────────────────────────────────────────┤
│ 主内容区                                                    │
│                                                              │
│  视角对应的大屏主画面                                       │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ 下钻分析层（抽屉 / 侧滑层 / 底部展开层）                    │
└──────────────────────────────────────────────────────────────┘
```

## 3. 顶部统一控制区线框图

### 3.1 功能目标

顶部控制区用于统一承载所有大屏层级都需要共享的控制能力，避免像当前一样分散在多张页面和多个局部工具栏中。

### 3.2 建议布局

```text
┌──────────────────────────────────────────────────────────────┐
│ Logo / 标题  | 模式切换 | 时间范围 | 空间层级 | 角色预设 | 刷新 │
│              |         |         |         |         | 状态   │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 字段说明

1. 左侧标题区：
   1. 固定标题“数据大屏”。
   2. 辅助副标题可根据当前视角变化，如“区域经营总览”“人员执行总览”。
2. 模式切换：
   1. 区域视角
   2. 人员视角
   3. 专题视角
3. 时间范围：
   1. 今日
   2. 近 7 天
   3. 近 30 天
   4. 自定义
4. 空间层级：
   1. 全国
   2. 浙江
5. 角色预设：
   1. 管理层
   2. 经营负责人
   3. 售前负责人
   4. 个人关注
6. 右侧状态区：
   1. 自动刷新状态
   2. 最后刷新时间
   3. 数据完整性提示

## 4. 区域视角页面蓝图

## 4.1 目标

区域视角用于回答：

1. 全国或浙江层级下，哪些区域业务更热。
2. 哪些区域存在经营缺口或风险。
3. 当前区域在不同统计口径下的业务结构如何。

区域视角的核心关系为：

区域 -> 各类统计口径

## 4.2 页面主结构

建议采用“上摘要、中地图、下分析”的三层结构。

```text
┌──────────────────────────────────────────────────────────────┐
│ A. 顶层摘要带                                               │
├──────────────────────────────────────────────────────────────┤
│ B1. 左联动区  │ B2. 地图主画布          │ B3. 右联动区       │
├──────────────────────────────────────────────────────────────┤
│ C. 底部多口径分析带                                          │
└──────────────────────────────────────────────────────────────┘
```

## 4.3 A. 顶层摘要带

### 内容

建议使用 6 到 8 个高价值摘要指标：

1. 客户总量
2. 项目总量
3. 项目金额
4. 合同金额
5. 售前活动数
6. 方案支撑数
7. 风险项目数
8. 异常区域数

### 呈现形式

1. 单值 KPI 卡
2. 环比/趋势箭头
3. 风险状态色标

### 作用

这一带用于先回答“总的怎么样”，然后再进入地图和具体区域分析。

## 4.4 B2. 地图主画布

### 功能职责

地图主画布承担以下职责：

1. 展示全国或浙江空间分布。
2. 作为区域切换入口。
3. 暴露热点、冷点与异常区域。
4. 提供下钻入口。

### 全国模式

1. 展示省级热力图。
2. 支持切换热力口径：客户、项目、金额、合同、活动、方案、风险。
3. 点击“浙江”进入浙江层级，点击其他省份不切换地图层级，而是在右侧展示该省统计明细。

### 浙江模式

1. 展示地市热力图。
2. 支持切换同样的热力口径。
3. 点击地市后联动该地市多口径分析。

### 重要说明

浙江模式必须使用真实热力图数据，不得回退为固定零值占位地图。

## 4.5 B1. 左联动区

左联动区建议作为“经营结构面板”。

### 建议模块

1. 区域客户分布图
2. 区域客户类型结构图
3. 区域项目阶段结构图

### 建议图表

1. 水平条形图
2. 环图
3. 漏斗图或分段柱图

### 作用

左侧优先回答“这个区域的盘子长什么样”。

## 4.6 B3. 右联动区

右联动区建议作为“风险与价值面板”。

### 建议模块

1. 区域项目金额排行
2. 区域合同金额排行
3. 区域风险对象榜
4. 区域售前活动趋势

### 建议图表

1. 排行条图
2. 风险气泡图
3. 折线图

### 作用

右侧优先回答“这个区域哪里最值钱、哪里最危险、哪里要继续看”。

## 4.7 C. 底部多口径分析带

底部是区域视角真正的价值区，用于一次性展示该区域在不同口径上的联动分析。

### 建议分为 4 个固定分析窗格

1. 客户窗格
2. 项目窗格
3. 方案窗格
4. 风险窗格

### 每个窗格建议内容

#### 客户窗格

1. 活跃客户数
2. 重点客户数
3. 客户推进状态分布
4. 客户排行

#### 项目窗格

1. 当前项目数
2. 金额分布
3. 项目阶段分布
4. 项目风险排行

#### 方案窗格

1. 当前方案数
2. 方案状态结构
3. 评审压力
4. 方案参与排行

#### 风险窗格

1. 风险对象总数
2. 高风险占比
3. 逾期事项数
4. 重点阻塞对象列表

### 下钻动作

区域视角的点击动作建议为：

1. 点区域：刷新区域联动分析
2. 点客户排行：打开客户侧滑分析层
3. 点项目排行：打开项目侧滑分析层
4. 点方案排行：打开方案侧滑分析层
5. 点风险对象：打开对象详情层

## 5. 人员视角页面蓝图

## 5.1 目标

人员视角用于回答：

1. 当前团队整体负载如何。
2. 哪些人过载，哪些人低活跃。
3. 每个人在做什么。
4. 每个人卡在哪些事项上。

人员视角的核心关系为：

人员 -> 事项及各类统计口径

## 5.2 页面主结构

建议采用“上摘要、中画像、下事项”的三层结构。

```text
┌──────────────────────────────────────────────────────────────┐
│ A. 团队摘要带                                               │
├──────────────────────────────────────────────────────────────┤
│ B1. 负载分布区 │ B2. 风险排行区 │ B3. 角色组对比区          │
├──────────────────────────────────────────────────────────────┤
│ C. 人员事项穿透区                                            │
└──────────────────────────────────────────────────────────────┘
```

## 5.3 A. 团队摘要带

### 建议指标

1. 管理中人员数
2. 当前任务总量
3. 当前逾期事项数
4. 高优事项数
5. 过载人员数
6. 连续低活跃人员数
7. 当前参与项目人数
8. 当前参与方案人数

### 呈现形式

1. 高密度 KPI 卡
2. 风险提醒标签
3. 环比趋势提示

## 5.4 B1. 负载分布区

### 建议图表

1. 人员负载分层堆叠图
2. 任务量分布柱图
3. 逾期分布柱图

### 核心问题

回答“整体人员负载是不是失衡”。

## 5.5 B2. 风险排行区

### 建议图表

1. 风险人员 Top N 排行图
2. 风险象限图

### 核心问题

回答“现在最应该盯谁”。

### 点击动作

点击某个人后，主页面下方的事项穿透区切换到该人的事项分析。

## 5.6 B3. 角色组对比区

### 建议图表

1. 角色组负载对比图
2. 角色组风险对比图
3. 角色组趋势图

### 核心问题

回答“不是单个谁的问题，而是哪一类岗位整体失衡”。

## 5.7 C. 人员事项穿透区

这是人员视角的最终落点，是二期最重要的交付之一。

### 建议结构

```text
┌──────────────────────────────────────────────────────────────┐
│ C1. 当前选中人员摘要                                         │
├──────────────────────────────────────────────────────────────┤
│ C2. 当前事项列表                                             │
├──────────────────────────────────────────────────────────────┤
│ C3. 当前事项详情 / 协同信息 / 时间线                         │
└──────────────────────────────────────────────────────────────┘
```

### C1. 当前选中人员摘要

展示：

1. 姓名
2. 角色
3. 当前任务数
4. 逾期事项数
5. 高优事项数
6. 最近活跃时间
7. 当前参与项目数
8. 当前参与方案数

### C2. 当前事项列表

事项列表需覆盖：

1. 任务
2. 待办
3. 方案评审事项
4. 项目协同事项

每条事项至少展示：

1. 事项名称
2. 类型
3. 所属客户
4. 所属项目
5. 所属方案
6. 优先级
7. 截止时间
8. 当前状态
9. 是否逾期

### C3. 当前事项详情

点击事项后，展开事项详情层，展示：

1. 事项完整描述
2. 当前阻塞原因
3. 协同对象
4. 最近推进时间
5. 操作时间线
6. 相关跳转入口

### 建议专题列表

1. 项目风险专题
2. 客户推进专题
3. 方案评审专题
4. 经营目标专题

### 建议布局

## 7. 下钻分析层蓝图

下钻分析层是整个二期页面体验统一性的关键。

建议采用统一规则：

1. 点击对象不优先跳整页。
2. 优先进入抽屉或侧滑分析层。
3. 如果需要跨模块跳转，再提供“进入详情页面”二次动作。

### 7.1 区域对象下钻

1. 点客户：打开客户分析抽屉
2. 点项目：打开项目分析抽屉
3. 点方案：打开方案分析抽屉
4. 点风险对象：打开统一风险详情抽屉

### 7.2 人员对象下钻

1. 点人员：切换下方事项穿透区
2. 点事项：打开事项详情抽屉
3. 点关联项目或方案：打开对象分析抽屉

## 9. 浙江地图专项线框要求

浙江地图专项虽然是数据问题，但在线框图阶段也要明确其页面责任。

### 9.1 浙江地图页面必须展示的内容

1. 地市热力图
2. 地市客户排行
3. 地市项目排行
4. 地市合同金额排行
5. 地市售前活动排行
6. 地市风险对象排行

### 9.2 浙江地图不得出现的情况

1. 进入浙江层后只剩一张空地图
2. 进入浙江层后只有客户数，其他口径全为 0
3. 进入浙江层后无法继续联动分析

## 10. 与当前实现的映射建议

当前已有能力可按如下方式吸收：

1. 当前地图中心舞台吸收到区域视角 B2。
2. 当前团队执行摘要、风险、角色/项目/客户/方案能力吸收到人员视角与专题视角。
3. 当前对象详情抽屉可改造成统一下钻分析层的基础组件。
4. 当前独立“团队驾驶舱”页面不再作为最终产品结构保留。

## 11. 设计评审时应重点回答的问题

在进入原型或视觉设计前，应先对以下问题达成一致：

1. 区域视角是否真正做到“区域 -> 多口径统计”而不是“只有地图”。
2. 人员视角是否真正做到“人员 -> 事项”而不是“只有排行”。
3. 团队执行能力是否被有效吸收到统一大屏，而不是保留割裂页面。
4. 浙江层级是否拥有独立业务价值，而不是全国地图的缩略版。
5. 首屏是否以图表为主，而不是重新堆出一个卡片墙。
