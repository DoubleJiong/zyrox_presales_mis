# 2026-04-08 数据大屏二期信息架构与页面分区草案

## 1. 文档目的

本草案用于明确数据大屏二期的产品定位、信息架构、页面分区、下钻方式与实施优先级，作为后续设计、开发、验收的统一依据。

本草案吸收以下判断：

1. 当前“团队驾驶舱”不应继续作为一个独立占据导航栏的模块存在，而应并入“数据大屏”内部。
2. 数据大屏二期的核心不是继续堆叠卡片，而是建立清晰的分析叙事、增强图表表达、统一区域与人员两条主视角。
3. 区域视角与人员视角是并列主视角，而不是“区域 -> 人员”的父子链路。
4. 浙江地图当前存在显示与数据口径双重问题，必须作为二期前置修复项处理。

## 2. 二期总体目标

数据大屏二期要解决的不是“再做一个更炫的页面”，而是把现有地图大屏、团队执行大屏以及后续人员统计诉求，整合为一张真正可用于管理决策和执行穿透的统一大屏。

二期完成后，数据大屏应同时满足以下两类核心诉求：

1. 区域视角：能够通过全国、浙江两个地图层级，观察各区域在客户、项目、金额、合同、售前活动、方案支撑、风险等统计口径下的业务状态。
2. 人员视角：能够观察所有管理中的人员当前工作状态，并继续下钻到每个人正在处理的具体事项。

## 3. 核心设计原则

### 3.1 一个入口，不再分裂为两个模块

一级导航只保留“数据大屏”。

“团队驾驶舱”不再作为一级导航项单独存在，而改造为数据大屏中的一个内部分析层，相关能力被吸收到“人员视角”和“专题视角”中。

### 3.2 双主视角并列

二期不采用“区域 -> 人员 -> 事项”的链式结构，而采用双主视角并列结构：

1. 区域视角：区域 -> 各类统计口径
2. 人员视角：人员 -> 事项及各类统计口径

两条视角共用一套基础数据口径与下钻框架，但分析目标不同。

### 3.3 地图是入口，不是全部

地图仍保留，但不再独占大屏中心叙事。

地图负责回答“哪里有问题、哪里有机会、哪里值得继续看”，而不是承担全部信息表达。地图之外必须配套高质量图表与排行分析。

### 3.4 图表优先，表格后置

大屏主层应以图表、热区、排行、趋势、结构分布为主。

表格保留，但只作为下钻层的补充，不作为首屏主体表达形式。

### 3.5 统一下钻模型

地图点击、排行点击、异常点击、对象点击，应尽量收敛到统一的下钻交互，而不是跳转到彼此割裂的独立页面。

## 4. 二期信息架构

建议将“数据大屏”内部重构为 3 个并列模式：

1. 区域视角
2. 人员视角
3. 专题视角

其中：

1. 区域视角承接全国/浙江地图和区域业务口径分析。
2. 人员视角承接当前团队执行驾驶舱的大部分能力，并继续强化到“个人事项”层。
3. 专题视角承接跨区域、跨人员的专项分析，例如项目风险、客户推进、方案评审、经营目标等。

## 5. 页面分区草案

## 5.1 顶部统一控制区

顶部统一控制区建议包含：

1. 模式切换：区域视角 / 人员视角 / 专题视角
2. 时间范围：今日、近 7 天、近 30 天、自定义
3. 空间层级：全国 / 浙江
4. 角色预设：管理层、经营负责人、售前负责人、个人视角
5. 数据刷新状态
6. 大屏主题状态与辅助说明

这一层替代当前分散在不同页面中的控制区与返回关系。

## 5.2 区域视角页面分区

区域视角建议分为三层：

### A. 顶层摘要带

展示当前选中层级下的关键经营指标：

1. 客户总量
2. 项目总量
3. 项目金额
4. 合同金额
5. 售前活动总量
6. 方案支撑总量
7. 风险项目数

### B. 中部地图主画布

地图仍然保留为中心视觉锚点，但职责聚焦在：

1. 热区呈现
2. 区域切换
3. 区域异常定位
4. 区域下钻入口

全国层级展示省级热力，浙江层级展示地市热力。

### C. 周边联动分析区

围绕地图联动展示区域统计图表，建议至少包含：

1. 客户分布排行
2. 项目金额排行
3. 合同金额排行
4. 售前活动趋势
5. 方案支撑结构
6. 风险分布图

说明：
当前区域视角的核心是“区域 -> 统计口径”，而不是“区域 -> 人员”。区域层如果出现人员信息，也应作为辅助手段，而不是主叙事。

## 5.3 人员视角页面分区

人员视角建议分为三层：

### A. 团队摘要带

展示团队级别的核心统计：

1. 在岗人员数
2. 当前任务总量
3. 当前逾期事项总量
4. 高优事项总量
5. 过载人员数
6. 连续低活跃人员数
7. 方案参与总量
8. 项目参与总量

### B. 人员画像图区

该区域替代当前大量纯卡片式展示，改为图表为主：

1. 负载分布图
2. 角色组对比图
3. 风险人员排行图
4. 人员趋势图
5. 人员协同热力图

### C. 人员事项穿透区

这是人员视角的最终落点，必须支持：

1. 看到每个人当前任务数
2. 看到每个人当前逾期事项
3. 看到每个人正在处理的具体事项
4. 看到事项所属客户、项目、方案
5. 看到事项的优先级、截止时间、阻塞原因、协同对象

说明：
人员视角的核心关系是“人员 -> 事项及各类统计口径”。

## 5.4 专题视角页面分区

专题视角用于承接跨区域、跨人员的横向专题，建议二期至少预留以下专题：

1. 项目风险专题
2. 客户推进专题
3. 方案评审专题
4. 经营目标专题

这些专题不要求全部在二期一次性完成，但页面结构应预留统一容器与切换入口。

## 6. 下钻策略

二期下钻采用“同视角内逐层深入”的原则。

### 6.1 区域视角下钻

区域视角下钻路径为：

1. 全国/浙江地图
2. 选中省份或地市
3. 查看该区域下各类统计口径
4. 继续下钻到客户、项目、方案、风险对象

区域视角的下钻目标是“理解该区域业务状态”，而不是切换成“人员主视角”。

### 6.2 人员视角下钻

人员视角下钻路径为：

1. 团队级人员统计
2. 选中个人
3. 查看个人各类统计口径
4. 继续下钻到个人事项列表与事项详情

人员视角的下钻目标是“理解个人工作状态与事项状态”。

## 7. 图表改造方向

当前团队执行页面图表不足，二期建议将现有内容图表化。建议的图表结构如下：

### 7.1 区域视角图表

1. 区域客户排行条形图
2. 区域项目金额排行图
3. 区域合同金额结构图
4. 区域售前活动趋势图
5. 区域方案支撑结构图
6. 区域风险分层图

### 7.2 人员视角图表

1. 人员负载分布图
2. 角色组对比图
3. 人员风险象限图
4. 人员事项趋势图
5. 协同热力矩阵

### 7.3 专题视角图表

1. 项目阶段漏斗图
2. 风险气泡图
3. 方案评审压力图
4. 客户推进路径图

## 8. 浙江地图专项问题

浙江地图应作为二期前置排障与修复项，不应继续视为一个普通优化点。

当前已发现的两个问题如下：

### 8.1 前端显示问题

当前浙江视角存在直接回退为空白占位数据的问题，前端在浙江视角下没有复用实际返回的热力图数据，而是直接返回固定的浙江城市零值数据。

这意味着即使后端返回了部分有效数据，页面也可能无法正确显示。

### 8.2 后端口径不足问题

当前浙江热力图主要依据客户地址中是否包含浙江城市名称来统计 customerCount，其他口径如：

1. projectCount
2. projectAmount
3. contractAmount
4. solutionUsage
5. preSalesActivity

仍然基本为空或未形成完整口径。

### 8.3 二期修复目标

浙江视角至少需要形成以下真实可展示口径：

1. 客户数
2. 项目数
3. 项目金额
4. 合同金额
5. 售前活动数
6. 方案支撑数
7. 风险项目数

同时需要明确：

1. 数据确实为空
2. 数据未接入
3. 数据已返回但前端未渲染

分别对应不同整改路径。

## 9. 与绩效管理的后续关系

本次二期不处理绩效管理模块整合，但需在架构上预留方向。

后续建议方向为：

1. 将绩效管理中“人员统计结果”逐步并入数据大屏的人员视角。
2. 独立“绩效管理”模块后续只保留绩效考核模型、规则、方案配置等能力。

该方向仅作为后续演进约束，不纳入本次交付范围。

## 10. 二期实施优先级建议

### P0：必须先完成

1. 收拢信息架构，取消团队驾驶舱独立一级导航方案
2. 修复浙江地图显示链路
3. 补齐浙江地图基础统计口径
4. 搭建统一大屏顶层框架

### P1：二期主体交付

1. 完成区域视角页面重构
2. 完成人员视角页面重构
3. 将团队执行现有能力吸收到人员视角
4. 完成统一下钻框架

### P2：增强项

1. 专题视角扩展
2. 高级图表增强
3. 动效与视觉强化
4. 更细的角色化视图编排

## 11. 二期完成标准

若二期完成，应达到以下标准：

1. 用户只通过“数据大屏”一个入口进入。
2. 用户可以清晰区分“区域视角”和“人员视角”。
3. 全国与浙江两个地图层级都能展示真实业务数据。
4. 用户能够在人员视角中看到个人事项落点，而不是停留在统计摘要。
5. 当前团队执行能力不再以独立页面存在，而是自然融入统一大屏中。
6. 页面主层图表表达明显增强，不再以表格与卡片堆叠为主。

建议在本草案基础上继续输出以下交付物：

2. 区域视角指标口径表
3. 人员视角指标口径表
4. 浙江地图数据专项排查清单
5. 二期实施任务拆解表
