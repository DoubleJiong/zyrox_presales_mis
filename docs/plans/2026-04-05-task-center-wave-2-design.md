# 工作台、日程、任务、消息中心模块演进计划第二波：任务中心视角收敛

日期：2026-04-05

状态：已完成

## 1. 第二波目标

第二波只处理任务中心视角收敛，不扩展到任务、待办、预警、消息的底层并表。

本波目标为：

1. 收敛 `/tasks` 与 `/my-tasks` 的双页面体验。
2. 保留 `/tasks` 作为唯一 canonical 任务页面。
3. 让“我的任务”变成任务中心中的个人视角，而不是另一套长期并行页面。
4. 明确“全部任务”“预警任务”“指派任务”的展示口径。

## 2. 变更分类

本波分类如下：

1. `requirement-gap`
2. `rule-conflict`

根因不是接口错误，而是任务中心与个人任务视角长期分裂，破坏了用户对任务模块的理解。

## 3. 第二波定版结论

### 3.1 Canonical 页面

1. `/tasks` 是唯一 canonical 任务页面。
2. `/my-tasks` 不再作为长期独立页面演进，仅保留兼容跳转职责。

### 3.2 视角划分

`/tasks` 内部保留两层视角：

1. `全部任务`：以项目任务为主，支持当前项目任务筛选、创建、编辑、删除。
2. `我的任务视角`：聚合当前用户的预警任务与指派任务。

其中“我的任务视角”再细分为：

1. `全部`
2. `预警任务`
3. `指派任务`

### 3.3 API owner

1. `/api/tasks` 仍是项目任务 canonical API。
2. `/api/my-tasks` 保留为个人任务混合读模型，供 `/tasks` 的个人视角复用。

## 4. 实施结果

1. `/tasks` 已接入个人任务视角，可在同一页面内切换“全部任务”和“我的任务”。
2. `/my-tasks` 已降级为跳转到 `/tasks?scope=mine` 的兼容别名。
3. 工作台“我的任务”入口已切换到 `/tasks?scope=mine`。
4. 任务中心顶部入口和说明文案已改为统一口径，不再继续强调两套平行页面。
5. `/api/tasks` 已补齐任务表主键序列漂移自修复，在 `bus_project_task_pkey` 冲突时会先重置序列，再重试一次任务创建。

## 5. 验证结果

1. 第二波 focused Playwright 正式回归通过：`tests/e2e/playwright/wave2-formal.spec.ts` 在 `5004` 端口校验了 `/tasks?scope=mine` 个人视角、任务创建、以及 `/my-tasks` 到 `/tasks?scope=mine` 的别名跳转。
2. 验证过程中已确认第二波唯一阻塞点是任务表主键序列漂移；修复后，正式运行中的 `5004` 服务已通过回归验证。

## 6. 当前结论

第二波已完成，任务中心的页面心智已经从“双页面分裂”收敛为“单页面双视角”。

1. 工作台任务统计口径与任务中心视角的进一步对齐。
2. 日程协作能力升级与任务/日程联动增强。
