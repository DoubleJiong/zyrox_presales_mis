# 第五阶段试点业务线选择说明

日期：2026-03-29

## 1. 结论

本轮第五阶段采用“客户/项目/方案可视主链路验证 - 投标审批 - 仪表盘核对”作为试点范围。

在真实业务陪跑验收开始前，必须先保证“客户/项目/方案”三条页面链路在业务账号下可正常进入、可见真实样本、可解释字段口径；否则不允许进入后续人工对账和签字。

由于当前机器无法连接真实业务数据库，且业务侧真实陪跑人员未在仓库环境中可获得，本轮执行采用“隔离环境影子试运行”方式先完成工程与治理验证，不直接宣布真实业务试点完成。

## 2. 试点范围

1. 登录与会话建立
2. 客户列表、项目列表、方案列表在业务账号下的可见性与返回契约验证
3. 项目可见性与无负责人项目隔离
4. 投标审批发起与统一审批实体联动
5. 方案评审审批桥接
6. 仪表盘核心指标口径返回
7. 身份生命周期与密码审计
8. 健康检查与试运行候选版本门禁

## 3. 参与角色映射

本轮影子试运行采用以下角色集合作为代表角色：

1. 系统管理员
2. 售前负责人 / 售前管理角色
3. 项目负责人
4. 审批人
5. 普通成员
6. 财务 / 商务角色

## 4. 样本来源

1. 自动化 API 与单元测试中的已固化主链路样本
2. 隔离 PostgreSQL 影子环境
3. 候选发布门禁执行结果

## 5. 风险说明

1. 本说明完成的是影子试运行选型，不替代业务侧真实试点负责人确认。
2. 在真实业务库不可连通前，不得把本说明解释为“真实样本已完成对账”。
