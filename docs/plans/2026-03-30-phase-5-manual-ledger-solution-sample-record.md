# 第五阶段台账项目方案样本生成记录

日期：2026-03-30

## 1. 目的

为“客户/项目/方案”主链路业务验收补齐方案样本，使业务账号可以在同一批台账导入项目上完成可视核对。

## 2. 样本来源规则

1. 来源对象：已导入的台账项目样本。
2. 选择条件：项目描述中包含 `原台账状态: 方案`。
3. 生成方式：为每个候选项目自动生成一条 `solutionCategory = project` 的项目方案，并写入项目方案关联表。
4. 幂等策略：使用稳定 `solutionCode`，重复执行时复用已存在方案，不重复插入。

## 3. 验收用途

1. 验证业务账号可进入方案模块并看到与台账项目对应的方案样本。
2. 支撑后续代表性样本抽检，形成“客户/项目/方案”三段闭环。
3. 该批样本仅用于第五阶段业务验收，不作为正式生产模板基线。

## 4. 本次执行结果

1. 预览命中候选项目 45 条，与台账中 `方案` 状态数量一致。
2. 实际导入新增项目方案 45 条，新增项目方案关联 45 条，无失败项。
3. 使用业务账号 `zhangwei@zhengyuan.com` 复验 `/api/solutions?pageSize=5`，返回总数 45。
4. 重复执行预览时全部命中已存在方案样本，验证了脚本幂等性。

## 5. 当前结论

1. “客户/项目/方案”三条主链路已具备业务可视验收入口。
2. 下一步应进入代表性样本抽检与总量对账，而不是继续停留在主链可见性修复。
