# 团队执行驾驶舱 V1 业务验收脚本

日期：2026-04-08

适用范围：团队执行驾驶舱 MVP 与 V1 轻量视角。

执行前准备：

1. 使用具备 `team-execution-cockpit:view` 权限的账号登录。
2. 确认本地或测试环境已准备可见的人员、项目、客户、方案样本。
3. 如需自动化复核，优先执行 `corepack pnpm run verify:team-execution-cockpit`。

## 1. 权限与入口

1. 打开导航，确认“团队执行驾驶舱”入口仅对有权限账号可见。
2. 直接访问 `/data-screen/team-execution`，确认有权限账号可进入页面。
3. 使用无权限账号访问同一路径，确认入口不可见且页面被拦截。

预期结果：

1. 权限边界与导航曝光一致。
2. 不影响数据大屏其他入口。

## 2. 首屏与统一筛选

1. 打开页面，确认汇总指标、风险热区、顶部控制区稳定渲染。
2. 切换 `角色视角 -> 项目视角 -> 客户视角 -> 方案视角`，确认页面不会闪崩或残留上一个视角内容。
3. 调整时间范围、快速筛选、关键词，确认 URL 中 `view/range/focus/q` 同步变化。
4. 刷新页面，确认筛选状态被保留。
5. 点击“重置”，确认页面回到默认 `role + 7d + all + 空关键词`。

预期结果：

1. 四类视角切换稳定。
2. URL 和页面状态一致。

## 3. 角色与项目视角

1. 在角色视角确认可见负载分布、角色组对比、人员风险排行、执行明细表。
2. 点击任一人员风险项，确认打开统一详情抽屉。
3. 在项目视角确认可见阶段分布、人力投入、风险热度、执行明细表。
4. 点击任一项目项，确认打开统一详情抽屉。

预期结果：

1. 抽屉信息与当前视角一致。
2. 人员与项目对象都能继续跳往详情页或任务中心。

## 4. 客户与方案视角

1. 在客户视角确认可见客户活跃度分布、事项规模排行、客户协同明细表。
2. 点击任一客户项，确认打开统一详情抽屉，且能跳转到客户详情页。
3. 在方案视角确认可见方案状态分布、评审压力排行、方案推进明细表。
4. 点击任一方案项，确认打开统一详情抽屉，且能跳转到方案详情页。

预期结果：

1. 最近互动、待评审、逾期评审、停滞时长等字段有值或有明确空值兜底。
2. 客户与方案视角口径可解释，没有明显漏数。

## 5. 统一详情抽屉与业务跳转

1. 分别从人员、项目、客户、方案四类入口打开详情抽屉。
2. 检查抽屉中的指标卡、详情字段、关联对象区块、动作按钮是否完整。
3. 点击“打开详情页”动作，确认能跳转到对应 canonical page。
4. 从人员或项目对象点击“进入任务中心”，确认 URL 带上正确的 `assigneeId` 或 `projectId`。
5. 在抽屉内点击关联对象，确认可以在抽屉内切换到新的对象详情。

预期结果：

1. 抽屉只读，不提供编辑入口。
2. 四类对象都能完成钻取。
